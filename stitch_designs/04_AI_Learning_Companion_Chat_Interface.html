Absolutely. A perceptron takes an input, multiplies it by a weight, adds a bias, and passes it through an activation function to determine the output. Here is a simple implementation using a step function as the activation function.

python 
import numpy as np

class Perceptron:
    def __init__(self, input_size, learning_rate=0.01):
        self.weights = np.zeros(input_size + 1)
        self.learning_rate = learning_rate

    def activation_fn(self, x):
        # Step activation function
        return 1 if x >= 0 else 0

    def predict(self, x):
        x = np.insert(x, 0, 1) # Insert bias unit
        weighted_sum = self.weights.T.dot(x)
        return self.activation_fn(weighted_sum)

The `activation_fn` here is a simple threshold. If the weighted sum is greater than or equal to 0, it fires (returns 1), otherwise it doesn't (returns 0). Does this clear things up?
